Public API of the Flowable engine.

Typical usage of the API starts by the creation of a {@link com.eudemon.flow.engine.ProcessEngineConfiguration} (typically based on a configuration file), from which a {@link com.eudemon.flow.engine.ProcessEngine} can be obtained.

Through the services obtained from such a {@link com.eudemon.flow.engine.ProcessEngine}, BPM and workflow operation can be executed:

{@link com.eudemon.flow.engine.RepositoryService}: Manages {@link com.eudemon.flow.engine.repository.Deployment}s.
{@link com.eudemon.flow.engine.RuntimeService}: For starting and searching {@link com.eudemon.flow.engine.runtime.ProcessInstance}s.
{@link com.eudemon.flow.engine.TaskService}: Exposes operations to manage human (standalone) {@link com.eudemon.flow.engine.task.Task}s, such as claiming, completing and assigning tasks.
{@link com.eudemon.flow.engine.IdentityService}: Used for managing {@link com.eudemon.flow.idm.api.User}s, {@link com.eudemon.flow.idm.api.identity.Group}s and the relations between them.
{@link com.eudemon.flow.engine.ManagementService}: Exposes engine administration and maintenance operations, which have no relation to the runtime exection of business processes.
{@link com.eudemon.flow.engine.HistoryService}: Exposes information about ongoing and past process instances.
{@link com.eudemon.flow.engine.FormService}: Access to form data and rendered forms for starting new process instances and completing tasks.
